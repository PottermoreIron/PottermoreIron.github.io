做题环节

感觉自己做算法题的能力一直是个很大的坑，但以后面试找工作又不得不搞，所以准备从今天开始，从头开始每天做三道LeetCode。
又到了每天愉快的做题环节。

两数之和

给定一个整数数组nums和一个整数目标值 target，请你在该数组中找出和为目标值target  的那两个整数，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
你可以按任意顺序返回答案。

作为LeetCode简单题的开门红，这道题不需要太多的想法。

One

所以我就直接一个暴力二维循环就解决了。
暴力总是富有美感的。
C++

Java

Python

Go

同时写四种语言自己老容易搞混，比如有些要加分号，有些不加分号，我甚至才知道Java双重循环内循环的变量属于外循环，我直接一个懵逼。不过确实，人生苦短，我选Python，只是效率不敢恭维。Go的话，还得适应一下它的写法。

Two

我掐指一算，算到了双指针。
如果有左右两个指针，一开始指向数组的第一个元素和最后一个元素，如果两者的和小于target，则左指针移动，如果两者的和大于target，则右指针移动，最后就这样用二分法逼近了target，既然只需要近似的一次循环，那么效率肯定不错。
只不过这个方法成立的根本是数组是有序数组，有序，才是我们如何移动指针的根本。既然库函数的排序效率肯定比我写的排序要高，那我就勉为其难直接用库函数排序吧。
但马上爷就歇菜了，排序完原来的index也没了，那爷还做个屁。
看来想要提升时间效率，就得求助于外来空间，我掐指一算又算到了map映射。
C++

Java

Python

Go

这种思路也很简单，主要是用一个map将value和index对应起来，再使用map自带的元素判断方法即可。
其中Python特殊一点，它不需要再新开辟一个map对象，因为这玩意儿列表自身的属性就已经满足了。爽是爽，慢是慢，两码事。

Three

其实第二种方法已经蛮不错了，只用了两次循环以线性的复杂度就完成了，但我们不止于此，实际上一次循环就可以完成，我们在这一次循环一边判断一边给map赋值，但注意的是，这样index在前面的数必将被找到，因为它们先被map保存。

两数相加

给你两个非空的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的，并且每个节点只能存储一位数字。
请你将两个数相加，并以相同形式返回一个表示和的链表。
你可以假设除了数字0之外，这两个数都不会以0开头。

这题也比较简单，直接一个模拟就行，给的是逆序的数，其实就方便进行加法进位。

1比特与2比特字符

有两种特殊字符：
第一种字符可以用一比特0表示。
第二种字符可以用两比特（10或11）表示给你一个以0结尾的二进制数组bits，如果最后一个字符必须是一个一比特字符，则返回true。

这道题蛮巧的，想通了就比较简单。
字符串以0结尾，而题目让我们判断这个字符串的最后一位是不是以一位字符结尾。
首先如果倒数第二位字符是0，那么必然该字符串以一位字符结尾，所以问题就出在倒数第二位字符是1我们如何考虑？
其实我们可以注意到，0不是重点，1才是重点。因为0百搭，既可以和1组成特殊字符，又可以自己就是特殊字符。
而我们应该关注连续的1，因为一旦有0，万物皆可成为特殊字符。
偶数个连续的1，运气好，则可以组成特殊字符11，如果是奇数个1，那么末尾就必有一个1和0搭伴。
